# 디자인 방향

스터디모임의 화면은 싸이월드 미니홈피 감성과 Discord식 커뮤니티 구조를 섞는 방향으로 잡았습니다. 과하게 화려한 화면보다는, 게시판 기능을 확인하기 쉬우면서도 프로젝트만의 분위기가 느껴지도록 구성했습니다.

## 핵심 방향

- 서비스 성격: 스터디 모집을 위한 커뮤니티 게시판
- 레이아웃: 좌측 카테고리, 중앙 모집글 목록/상세, 우측 보조 정보
- 분위기: 싸이월드식 미니룸/마이룸 감성 + Discord식 빠른 카테고리 탐색
- 표현 방식: 신입 프로젝트답게 기능 흐름이 잘 보이도록 구성
- 디자인 강도: 비어 보이지 않게 패널과 배경을 사용하되, 기능을 가리지 않도록 조절

## 주요 화면

### 메인

- 좌측에는 카테고리와 MY ROOM을 배치했습니다.
- 중앙에는 검색, 모집글 목록, 페이지네이션 흐름을 배치했습니다.
- 우측에는 접속 멤버, 추천 기능, 로드맵 진행도를 배치했습니다.

### 모집글 상세

- 모달 형태로 열어 메인 화면 흐름을 유지합니다.
- 상세 모달 내부 스크롤을 분리해 배경 페이지가 같이 움직이지 않도록 했습니다.
- 작성자 프로필 캐릭터, 본문, 반응 버튼, 댓글/답글을 한 화면 안에서 확인할 수 있게 구성했습니다.

### MY ROOM

- 로그인 전에는 방문자 상태를 보여줍니다.
- 로그인 후에는 사용자 닉네임과 캐릭터를 보여줍니다.
- 캐릭터는 배경, 몸통, 얼굴, 손발, 모자, 도구, 동작을 선택할 수 있게 했습니다.

### 관리자 페이지

- 일반 사용자 메뉴에 관리자 진입 버튼을 노출하지 않았습니다.
- 별도 주소로 들어가 관리자 로그인을 하도록 구성했습니다.
- 회원, 모집글, 댓글, 신고, 작업 로그를 탭 형태로 나누어 한 번에 너무 많은 데이터가 보이지 않도록 했습니다.
- 모집글은 목록에서 바로 삭제하지 않고 미리보기 모달을 거쳐 확인 후 처리하도록 했습니다.

## UI 개선 과정

- 로그인/회원가입 메뉴를 하나의 인증 모달로 통합했습니다.
- 로그인 상태에 따라 MY ROOM과 권한 버튼이 달라지도록 정리했습니다.
- 본인 글과 본인 댓글에만 수정/삭제 버튼이 보이도록 했습니다.
- 좋아요, 북마크, 모집 상태 버튼에는 서로 다른 색상과 이펙트를 적용했습니다.
- 연속 클릭으로 상태가 꼬이지 않도록 버튼 쿨타임을 적용했습니다.
- 신고 완료, 삭제 완료, 제재 완료는 브라우저 기본 alert 대신 서비스 분위기에 맞춘 알림 UI로 표시했습니다.

## 이후 개선할 점

- 모바일 화면에서 미니홈피 감성을 유지하면서도 목록 가독성을 더 높이기
- 캐릭터 아이템을 더 정리된 프리셋으로 제공하기
- 관리자 페이지에서 데이터가 많아졌을 때 페이지네이션과 정렬 조건 강화하기
