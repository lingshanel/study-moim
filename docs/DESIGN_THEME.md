# 스터디모임 디자인 방향

스터디모임은 싸이월드 감성에서 출발하되, 카테고리와 커뮤니티 구조는 Discord처럼 빠르게 이동할 수 있는 형태로 잡습니다.

## 방향

- 서비스 톤: 레트로 커뮤니티 감성의 스터디 모집 서비스
- 화면 구조: 왼쪽 카테고리, 가운데 모집글 목록/상세, 오른쪽 보조 정보
- 표현 방식: 신입 프로젝트답게 과하게 보이기보다 직접 구현한 기능 흐름이 잘 보이도록 구성
- 디자인 강도: 빈 화면처럼 보이지 않게 패널, 배지, 작은 상태값을 사용하되 장식이 기능을 가리지 않도록 조절

## 주요 화면

- 홈: 모집글 목록, 카테고리, 검색, 페이지네이션
- 로그인 / 회원가입: 단순한 폼과 안내 문구
- 모집글 상세: 본문, 작성자, 댓글, 모집 상태
- 모집글 상세는 목록 옆 고정 패널보다 싸이월드 미니홈피 방처럼 뜨는 모달 흐름으로 구성
- 오른쪽 보조 패널은 댓글보다 빠른 메뉴, 오늘의 스터디, 추천 기능처럼 탐색을 돕는 영역으로 사용
- 글쓰기 / 수정: 제목, 내용, 카테고리, 진행 방식, 모집 인원
- 마이페이지: 내 정보, 내가 쓴 글, 북마크한 글
- 관리자: 사용자/게시글 관리 기능은 3차 범위에서 추가

## MVP 적용 순서

### 1차

- 회원가입
- 로그인 / 로그아웃
- 내 정보 조회
- 카테고리 목록
- 모집글 CRUD
- 본인 모집글만 수정/삭제
- 댓글 CRUD
- 검색
- 페이지네이션

### 2차

- 좋아요
- 북마크
- 모집 상태 변경
- 비밀번호 변경
- 회원 탈퇴

### 3차

- 이미지 업로드
- 관리자 기능
- 로그인 실패 제한
